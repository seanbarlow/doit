## Overview

The `/doit.roadmapit` command creates and manages project roadmaps with prioritized requirements, deferred functionality, and AI-suggested enhancements. It provides guided prompts to help users articulate their project vision and maintains a structured approach to requirement management.

## Technical Details

### Command Structure

The command supports multiple operations:

| Pattern | Operation | Example |
|---------|-----------|---------|
| `create [vision]` | Create new roadmap | `/doit.roadmapit create a task management app` |
| `add [item]` | Add item to roadmap | `/doit.roadmapit add user authentication` |
| `defer [item]` | Move item to deferred | `/doit.roadmapit defer social login` |
| `reprioritize` | Review and change priorities | `/doit.roadmapit reprioritize` |
| (empty) | Interactive update | `/doit.roadmapit` |

### File Locations

- Command: `.claude/commands/doit.roadmapit.md`
- Roadmap template: `.doit/templates/roadmap-template.md`
- Archive template: `.doit/templates/completed-roadmap-template.md`
- Runtime storage: `.doit/memory/roadmap.md` and `.doit/memory/completed_roadmap.md`

### Integration Points

1. **Scaffoldit**: New projects include the roadmapit command (10 commands total)
2. **Checkin**: Completed features trigger archiving of matching roadmap items
3. **Constitution**: AI suggestions align with project principles when constitution exists
