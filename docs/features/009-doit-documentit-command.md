## Overview

Added `/doit.documentit` command for documentation management (organize, index, audit, cleanup, enhance-templates) and enhanced `/doit.scaffoldit` to generate `.doit/memory/tech-stack.md` capturing technology choices at scaffold time.

## Key Features

### New `/doit.documentit` Command

A documentation management command with 5 subcommands:

| Subcommand | Description |
|------------|-------------|
| `organize` | Reorganize scattered documentation into standard structure |
| `index` | Generate or update `docs/index.md` with categorized links |
| `audit` | Check documentation health (broken links, missing headers, coverage) |
| `cleanup` | Find duplicate and orphaned documentation files |
| `enhance-templates` | Suggest improvements for command templates |

### Enhanced `/doit.scaffoldit`

- Now generates `.doit/memory/tech-stack.md` during project scaffolding
- Documents languages, frameworks, key libraries with versions
- Records architecture decisions with rationale
- Updated command count from 10 to 11 commands

## Technical Details

- Uses markdown-based slash commands (no src/ code)
- File-based storage in `.doit/memory/`, `.doit/templates/`, and `docs/`
- Auto-generated markers pattern for preserving manual content
- Template distribution: command -> .doit/templates/commands/ -> templates/commands/

## Usage

```bash
# Show interactive menu
/doit.documentit

# Organize scattered documentation
/doit.documentit organize

# Generate documentation index
/doit.documentit index

# Audit documentation health
/doit.documentit audit

# Find duplicates and orphans
/doit.documentit cleanup

# Improve command templates
/doit.documentit enhance-templates
```
